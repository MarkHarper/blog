The Week Six and Seven Recap

23 June 2015

It's been a long time since the last update. The work at The Iron Yard has been a bit crazy over the last two weeks. A group project with the Rails team during week six proved to be both challenging and time-consuming. Then, in week seven we took a bit of a breather with review and an introduction to Backbone Js. Here is a short list of the topics we have covered in the past two weeks.

  • Browserify - You will learn to love browserify, even if it confuses you the first time around. It helps you organize the dependencies of each file in your app.
  • Susy Grid System - This is easily the best grid system that we have been exposed to in the program.
  • Jasmine - I really want to use this javascript testing library to drive the development of future projects. Test driven development is a very different approach from the plan/guess and check method that I've been using.
  • Regular Expressions - At long last, we learned Regex. These brief lines of gibberish make parsing strings so much easier.
  • Parsley Js - This is a validation library that is very handy. Although, it can be difficult to implement in browserify & gulp.
  • Ruby on Rails - A bit of a disclaimer: we clearly didn't learn all of Ruby on Rails, but we did learn how to work with Rails developers as Front End developers. It still amazes me the way that Rails magically builds the framework of an application.
  • Pure CSS - This is a very easy to use css framework, with emphasis on 'very easy'. The aesthetic is very much like bootstrap, but it much smaller and easier to customize.
  • Backbone Js - Our very first JS framework. At this point, I'm still a bit thrown by the way everything comes together in Backbone, but I'm catching on. One thing is clear. Backbone adds a ton of power to our development arsenal.
Takeaways

-Integrating with Rails is half the battle : Getting styles and html running in Rails truly was a 2 day battle. Here is a quick rundown of the primary issue. Rails uses ERB files to produce HTML and pull in data from models. The Rails class had not covered adding classes to generated elements yet. So, we had to work together to add classes for our CSS to target.
-Commit early and commit often : Another problem that we encountered on the Rails project was merging. My Front End teammate and I styled the project in a separate branch. This required us to change the ERB templates. As our Rails teammate added new features on the backend, our branches gained more and more conflicting changes. With one day left on the project, we had to spend a few hours coming up with a working master branch. On your future projects, I implore you to commit early and commit often. Use Git as it was designed.

Resources:

  • An abridged version of Javascript: The Good Parts - I've heard a great deal about Javascript: The Good Parts. This overview on Github seemed like a valuable find.

  • You Don't Know Js - Here is great refresher series on vanilla Javascript.

  • Trello - As I was adding some final stylings to the Rails project, I created a todo list on Trello. It is a great way to keep track of your project, especially if you tend to jump from feature to feature the way that I do.
Fascinating Tech News:

  • 7 future web design trends - This article covers several of the design concepts that I have picked up on during this course.

  • Apple should just buy DuckDuckGo - If Apple is as serious about privacy as they claimed to be at the most recent WWDC, then this would be an exciting way to prove it.

Dream Destination of the Week

Havana, Cuba

As some of you may know, the US has just reopened direct travel to Cuba. So, this beautiful city was an easy choice this week. If the food in Havana is as good the Cuban food I've enjoyed in the States (as I'm sure it is), then I will definitely be heading there in the near future.
